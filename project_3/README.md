## Описание проекта

Этот проект выполнен в рамках обучения на курсе SkillFactory Data Science. 

**Суть проекта:** Построить модель машинного обучения, которая предсказывает рейтинг отеля на основе отзывов. Такая модель помогает выявлять отели, которые могут накручивать себе рейтинг (если реальная оценка сильно отличается от предсказанной).

## Постановка задачи

Компания Booking.com сталкивается с проблемой нечестных отелей, которые искусственно завышают свой рейтинг. Один из способов борьбы - построить модель, предсказывающую ожидаемый рейтинг отеля. Если предсказание модели сильно отличается от фактической оценки - это сигнал для проверки отеля.

**Цель:** Обучить модель, которая по имеющимся данным об отзыве предсказывает оценку рецензента (`reviewer_score`).

**Задачи:**
- Очистить и подготовить данные
- Создать новые признаки (feature engineering)
- Обучить модель RandomForestRegressor
- Оценить качество модели (метрика MAPE)
- Сделать сабмит на Kaggle



## Данные

### Источник данных
Датасет взят с https://www.kaggle.com/ в рамках закрытого соревнования на Kaggle: [SF-DST] Booking reviews.

### Описание полей

| Поле | Описание |
|------|----------|
| `hotel_address` | Адрес отеля |
| `review_date` | Дата размещения отзыва |
| `average_score` | Средний балл отеля за последний год |
| `hotel_name` | Название отеля |
| `reviewer_nationality` | Страна рецензента |
| `negative_review` | Текст отрицательного отзыва |
| `review_total_negative_word_counts` | Количество слов в отрицательном отзыве |
| `positive_review` | Текст положительного отзыва |
| `review_total_positive_word_counts` | Количество слов в положительном отзыве |
| `reviewer_score` | Оценка рецензента (целевая переменная) |
| `total_number_of_reviews_reviewer_has_given` | Сколько отзывов оставил рецензент ранее |
| `total_number_of_reviews` | Общее количество отзывов об отеле |
| `tags` | Теги, которые рецензент дал отелю |
| `days_since_review` | Количество дней с момента отзыва |
| `additional_number_of_scoring` | Количество оценок без текстового отзыва |
| `lat` | Географическая широта отеля |
| `lng` | Географическая долгота отеля |

### Размер данных
- Обучающая выборка: ~500 000+ строк
- Тестовая выборка: ~96 756 строк

## Этапы работы

### 1. Предварительный анализ данных (EDA)

- Определены типы данных в каждом столбце
- Выявлены строковые столбцы (`object`): `hotel_address`, `hotel_name`, `reviewer_nationality`, `negative_review`, `positive_review`, `tags`, `days_since_review`
- Числовые столбцы: `additional_number_of_scoring`, `average_score`, `review_total_negative_word_counts`, `review_total_positive_word_counts`, `total_number_of_reviews`, `total_number_of_reviews_reviewer_has_given`, `lat`, `lng`

### 2. Очистка данных

**Первый подход (базовый):**
- Удалены все строковые столбцы
- Пропущенные значения заполнены нулями
- Результат: MAPE = 0.1413 (14.13%)

### 3. Разведывательный анализ

В процессе работы были исследованы:

- **Уникальные отели:** 1492 уникальных названия
- **Даты отзывов:** первый отзыв - 2015-08-04, последний - 2017-08-03
- **Теги:** 2368 уникальных тегов, самый популярный - `'Leisure trip'` (313 593 упоминания)
- **Продолжительность проживания:** чаще всего путешественники останавливаются на **1 ночь** (145 373 отзыва)

### 4. Результаты
Базовое решение (только числовые признаки)	0.1413

### 5. Выводы

- **Чистка данных - ключевой этап:** Модель не работает со строковыми данными и пропусками. Их правильная обработка напрямую влияет на качество предсказаний.
- **MAPE 14.13%** это хороший стартовый результат. Дальнейшее улучшение возможно через создание новых признаков и более глубокую обработку текстовых данных.
- **Разведывательный анализ помогает:** Изучение тегов, дат и геолокации даёт понимание, какие дополнительные признаки могут быть полезны.
