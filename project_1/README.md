# Прогнозирование уровня заработной платы для соискателей | Data Science проект для HeadHunter

**Проект выполнен в рамках отработки навыков Data Science на реальных данных.**

##  О проекте

Компания **HeadHunter** столкнулась с проблемой: значительная часть соискателей не указывает желаемую заработную плату при заполнении резюме. Это усложняет процесс подбора для рекрутеров и снижает эффективность платформы.

**Цель проекта** — выполнить полный цикл предобработки и анализа данных, чтобы подготовить базу резюме для построения модели машинного обучения. Эта модель в дальнейшем сможет автоматически рекомендовать уровень заработной платы, исходя из информации, указанной соискателем в резюме.

##  Основные задачи

Проект разбит на четыре логических этапа, отражающих стандартный пайплайн работы с данными:

1.  ** Базовый анализ структуры данных**
    *   Первичное знакомство с датасетом
    *   Оценка качества и полноты данных
    *   Формулирование гипотез для дальнейшего исследования

2.  ** Преобразование данных**
    *   Приведение данных к согласованным типам (`int`, `float`, `datetime`)
    *   Обработка аномалий и выбросов в ключевых полях
    *   Стандартизация и очистка текстовых полей (специализация, локация)

3.  ** Разведывательный анализ (EDA)**
    *   Анализ распределения целевой переменной — заработной платы
    *   Исследование взаимосвязи зарплаты с ключевыми факторами: опыт работы, образование, локация, специализация
    *   Визуализация результатов и формулирование выводов

4.  ** Очистка данных (Data Cleaning)**
    *   Удаление дубликатов и нерелевантных записей
    *, готового к построению моделей машинного обучения

##  Используемый стек технологий

*   **Язык программирования:** Python 3
*   **Основные библиотеки:**
    *   `pandas`, `numpy` — для манипуляций с данными
    *   `matplotlib`, `seaborn`, `plotly` — для визуализации и EDA
    *   `scikit-learn` — для предобработки и будущего моделирования

##  Ключевые этапы работы и выводы

### 1. Базовый анализ
*   **Обзор данных:** Проанализирована структура датасета, определены типы данных и наличие пропусков.
*   **Первые гипотезы:** Сформулированы предположения о влиянии города, опыта работы и образования на уровень зарплаты.

### 2. Преобразование данных
*   **Стандартизация зарплаты:** Значения в столбце "Зарплата" приведены к числовому формату, удалены нерелевантные символы и единицы измерения.
*   **Обработка дат:** Поля с датами (например, "Обновлено резюме") преобразованы в тип `datetime` для анализа временных трендов.
*   **Чистка текста:** Унифицированы названия городов и специализаций, что позволило корректно группировать данные.

### 3. Разведывательный анализ (EDA)
*   **Распределение зарплат:** Построены гистограммы и box-plot, выявлен диапазон "средних" зарплат и основные выбросы.
*   **Влияние факторов:**
    *   **Опыт работы:** Обнаружена четкая положительная корреляция между стажем и уровнем дохода.
    *   **Образование:** Выявлено, что соискатели с высшим образованием в среднем имеют более высокие зарплатные ожидания.
    *   **Локация:** Подтверждена гипотеза о значительной разнице в зарплатах между Москвой, Санкт-Петербургом и другими регионами.
    *   **Специализация:** Определены наиболее высокооплачиваемые области (например, Data Science, Разработка) и наименее оплачиваемые.

### 4. Очистка данных
*   **Борьба с пропусками:** Для категориальных признаков использовано заполнение модой, для числовых — медианой.
*   **Удаление дубликатов:** Устранены полные дубликаты записей.
*   **Обработка выбросов:** Для целевой переменной применены статистические методы (IQR) для "мягкого" ограничения разумного диапазона.

##  Используемые данные

В проекте используется анонимизированная выгрузка резюме с платформы hh.ru. Данные предоставлены для учебных целей и содержат информацию о желаемой зарплате, опыте работы, образовании, локации и специализации соискателей.

##  Итоги

Проект представляет собой **полноценный пример работы аналитика/дата-сайентиста** на реальной бизнес-задаче от HeadHunter.
В результате выполнения всех этапов был получен **очищенный и структурированный датасет**, полностью готовый для передачи на этап feature engineering и построения прогнозной модели (регрессии) для автоматического определения уровня заработной платы.

Этот проект — важная часть портфолио, демонстрирующая навыки:
*   Глубокой предобработки и очистки "сырых" данных.
*   Системного разведывательного анализа с формулировкой бизнес-выводов.
*   Работы с реальной бизнес-задачей и данными.
