### Описание проекта    
Разработка алгоритма, который угадывает загаданное компьютером число в диапазоне от 1 до 100 за минимальное количество попыток. Проект демонстрирует разницу между наивным случайным поиском и оптимизированным алгоритмическим подходом. Решение задания 8.7 находится в файле "major_task.py"

:arrow_up:[к оглавлению](#Оглавление)

### Какой кейс решаем?    
Нужно написать программу, которая угадывает число за минимальное число попыток

**Условия соревнования:**  
- Компьютер загадывает целое число от 1 до 100, и нам его нужно угадать
- Алгоритм учитывает информацию о том, больше ли случайное число или меньше нужного нам
- Решение должно быть эффективным и работать за минимальное среднее количество попыток

**Метрика качества**     
Результаты оцениваются по среднему количеству попыток при 1000 повторений. Чем меньше среднее количество попыток - тем лучше алгоритм.

**Что практикуем**     
- Учимся писать хороший код на Python
- Изучаем эффективность разных подходов к решению задач
- Работаем с библиотекой NumPy для научных вычислений

:arrow_up:[к оглавлению](#Оглавление)

### Краткая информация о данных
В проекте используются синтетические данные:
- Диапазон чисел: от 1 до 100 включительно
- Тестовая выборка: 1000 случайных чисел
- Генерация чисел: с помощью `np.random.randint(1, 101)`
- Оценка качества: среднее арифметическое количество попыток

:arrow_up:[к оглавлению](#Оглавление)

### Этапы работы над проектом  
1. **Реализация наивного алгоритма** - случайный поиск без использования дополнительной информации
2. **Анализ эффективности** - оценка среднего количества попыток (≈95 попыток)
3. **Разработка оптимизированного алгоритма** - реализация бинарного поиска
4. **Тестирование и сравнение** - проверка эффективности нового алгоритма
5. **Документирование результатов** - оформление выводов и рекомендаций

:arrow_up:[к оглавлению](#Оглавление)

### Результаты  
Были реализованы и протестированы два алгоритма:

**1. Наивный алгоритм (random_predict):**
- Стратегия: случайные догадки
- Среднее количество попыток: ≈95
- Максимальное количество попыток: >100

**2. Алгоритм бинарного поиска (needed_predict):**
- Стратегия: последовательное деление диапазона пополам
- Среднее количество попыток: ≈6
- Максимальное количество попыток: 7

**Улучшение эффективности:**
- Снижение среднего количества попыток в **16 раз**
- Гарантированное нахождение числа не более чем за 7 попыток
- Стабильность результатов независимо от загаданного числа

### Выводы  
1. **Алгоритмический подход имеет решающее значение** - правильный выбор алгоритма может улучшить производительность на порядок
2. **Бинарный поиск оптимален** для задач поиска в отсортированных диапазонах
